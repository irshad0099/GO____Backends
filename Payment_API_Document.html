Section 03

Passenger — Ride Payment Flow

Wallet, Cash, UPI, Card — sabka flow

3.1 Wallet Payment

Frontend ko kuch nahi karna — Ride complete hote hi backend automatically wallet debit karta hai.

Ride request mein sirf payment_method pass karo:

RIDE REQUEST BODY
{
  "pickup_latitude": 28.6139,
  "pickup_longitude": 77.2090,
  "dropoff_latitude": 28.5355,
  "dropoff_longitude": 77.3910,
  "vehicle_type": "auto",
  "payment_method": "wallet"
}

Agar balance kam hai — ride request fail hogi:

ERROR RESPONSE
{
  "success": false,
  "message": "Insufficient wallet balance. Required: ₹150.00, Available: ₹80.00"
}

3.2 Cash / Personal UPI Payment

Passenger side pe koi extra API call nahi. Sirf ride request mein method pass karo — baki driver confirm karega.
RIDE REQUEST BODY
{
  "payment_method": "cash"
}

3.3 Online Payment — UPI / Card (Razorpay)

Ride complete hone ke baad 3 steps follow karo:

Step 1 — Create Payment Order

POST /payments/orders
REQUEST BODY
{
  "purpose": "ride_payment",
  "amount": 150.00,
  "payment_method": "upi",
  "payment_gateway": "razorpay",
  "ride_id": 789
}
RESPONSE
{
  "success": true,
  "data": {
    "order": { "orderNumber": "PAY20260506XYZ", "amount": 150, "status": "pending" },
    "requiresGateway": true,
    "gatewayOrderId": "order_RZP456"
  }
}

Step 2 — Open Razorpay SDK

// Flutter example
Razorpay razorpay = Razorpay();
var options = {
  'key': 'rzp_live_XXXXXX',
  'order_id': 'order_RZP456',   // gatewayOrderId from Step 1
  'amount': 15000,               // in paise
  'name': 'GO Mobility',
  'description': 'Ride Payment',
};
razorpay.open(options);

Step 3 — Verify Payment

POST /payments/verify
REQUEST BODY
{
  "razorpay_order_id": "order_RZP456",
  "razorpay_payment_id": "pay_ABC123",
  "razorpay_signature": "hash_from_razorpay_callback"
}
✅ Verify success ke baad driver wallet automatically credit ho jaata hai (80% share).

Driver — Cash Collection (Ride)

Jab passenger cash ya personal UPI se payment kare

POST Collection Confirm

POST /rides/:rideId/collect-confirm
Kab call karo: Ride complete hone ke baad jab passenger ne cash ya personal UPI se payment kiya ho.
REQUEST BODY — Cash
{ "method": "cash" }
REQUEST BODY — Personal UPI
{ "method": "personal_upi" }
RESPONSE
{
  "success": true,
  "message": "Cash collection confirmed",
  "data": {
    "rideId": 789,
    "method": "cash",
    "finalFare": 150.00,
    "netEarnings": 120.00,
    "platformFee": 30.00,
    "paymentStatus": "collected_by_driver",
    "confirmedAt": "2026-05-06T11:30:00Z"
  }
}

Socket Events (After Confirmation)

EventKaun Receive KareData
ride:payment_settledPassengerrideId, amount, method, message
driver:earnings_creditedDriverrideId, netEarnings, method, walletUpdated
ride:payment_settled (Passenger receives)
{
  "rideId": 789,
  "amount": 150.00,
  "method": "cash",
  "platformFee": 30.00,
  "message": "Driver confirmed cash payment received"
}
driver:earnings_credited (Driver receives)
{
  "rideId": 789,
  "netEarnings": 120.00,
  "method": "cash",
  "walletUpdated": true
}

Backend mein kya hota hai confirm ke baad

1
Ride validateStatus = completed, method = cash/personal_upi
↓
2
Fare split calculatefinalFare × 20% = platformFee, 80% = netEarnings
↓
3
Driver wallet creditnetEarnings (₹120) wallet mein credit
↓
4
cash_balance updateplatformFee (₹30) driver ke cashBalance mein add
↓
5
Socket notifyPassenger aur Driver dono ko notify
