GoMobility — Driver API Guide

Ye document sirf driver app ke liye hai. Har API ka — kab call karo, kyun karo, kya bhejo, kya milega — sab yahan clearly likha hai. Frontend developer ko yahi doc follow karni hai.

🔗 Base URL & Authentication
Base URL: https://your-domain.com/api/v1
🔑 Har API mein Authorization header dena zaroori hai (except login/register)

Authorization: Bearer <your_jwt_token>

Token login/OTP verify ke baad milta hai. Token expire hone par refresh karo.
HeaderValueRequired
Authorization Bearer <jwt_token> ✓ Sabhi APIs mein
Content-Type application/json POST/PUT/PATCH mein
⚡ Socket Events — Driver ko kaunse events sunne hain
Driver app login karne ke baad socket connect karo. Auth karo: socket.emit('auth:login', { userId, role: 'driver' }). Phir in events ko listen karo:
ride:new_requestLISTEN
Nayi ride request aane par ye event milta hai. Driver ko accept/reject karna hoga 30 seconds mein.
Data jo milega
{
  "rideId": 123,
  "passenger": { "name": "Rahul", "rating": 4.5 },
  "pickup": { "address": "Connaught Place, Delhi", "lat": 28.63, "lng": 77.22 },
  "drop": { "address": "Hauz Khas, Delhi", "lat": 28.54, "lng": 77.20 },
  "estimatedFare": 120,
  "paymentMethod": "cash",  // cash | wallet | upi | card | corporate
  "distanceKm": 5.2,
  "vehicleType": "auto",
  "timeoutSeconds": 30
}
driver:cash_dues_updatedLISTEN
Cash ride complete hone ke baad driver ko ye event milta hai. Screen pe pending deposit dikhao.
Data jo milega
{
  "rideId": 123,
  "netEarnings": 102,         // Driver ka share
  "platformShareDue": 18,      // Company ko dena hai
  "message": "Deposit ₹18 to receive your ₹102 earnings"
}
driver:earnings_creditedLISTEN
Online/wallet ride mein driver ka wallet credited hone par ye event aata hai.
Data jo milega
{
  "rideId": 123,
  "netEarnings": 102,
  "walletBalance": 350.50,
  "paymentMethod": "wallet"
}
ride:otp_verifiedLISTEN
OTP verify hone ke baad ride in_progress hoti hai — ye confirm karta hai.
{ "rideId": 123, "verifiedAt": "2024-01-01T10:00:00Z", "message": "OTP verified. Ride started!" }
ride:status_changedLISTEN
Ride ka status change hone par milta hai (dono sides ko).
{ "rideId": 123, "status": "in_progress", "timestamp": "..." }
driver:location_updateEMIT
Driver apni location bhejta hai (har 3-5 seconds). Ride active hai tab ride:join karna zaroori hai tracking ke liye.
// Ride join karo (accept karne ke baad):
socket.emit('ride:join', { rideId: 123 });

// Har 3-5 sec location bhejo:
socket.emit('driver:location_update', {
  latitude: 28.63,
  longitude: 77.22,
  rideId: 123  // active ride mein
});
driver:availability_toggleEMIT
Driver online/offline toggle karne par emit karo.
socket.emit('driver:availability_toggle', { isAvailable: true });
💳 Payment Methods — Driver ko kya karna hota hai
💵 Cash
Passenger driver ko physically cash deta hai.
Driver kya kare: Cash lene ke baad collect-confirm API call karo. Platform fee baad mein deposit karni hogi.
📱 Personal UPI
Passenger driver ke personal UPI pe pay karta hai (driver ka own UPI).
Driver kya kare: Payment receive hone ke baad collect-confirm method=personal_upi se call karo.
👛 Wallet / Card / Online UPI
Passenger app wallet, card ya online UPI se pay karta hai — payment automatic process hota hai.
Driver kya kare: Kuch nahi karna. Earning automatically wallet mein credit ho jaati hai.
🏢 Corporate
Corporate account se payment hoti hai (business rides).
Driver kya kare: Ride complete karo. Payment backend handle karta hai.
⚠️ Cash/Personal UPI rides mein earning immediately wallet mein NAHI aati. Driver ko platform fee (company ka hissa) deposit karna hoga. Tab hi apni earning wallet mein milegi (Rapido model).
🗺️ Complete Ride Flow — Step by Step
Standard Ride Flow (Driver Side)
1
App Open → Online Ho Jao

Driver app open karo, location update karo, phir online toggle karo.

PATCH /api/v1/drivers/availability PUT /api/v1/drivers/location
2
Nayi Ride Request Aai

Socket event milega — 30 seconds ka timer dikhao. Driver accept ya reject kar sakta hai.

LISTEN: ride:new_request
3
Ride Accept Karo

Accept button press karo. Ride join karo socket se. Location updates shuru karo.

POST /api/v1/rides/:rideId/accept EMIT: ride:join {rideId}
4
Pickup Pe Pohoncho → OTP Generate Karo

Pickup location pe pahunchne ke baad OTP generate karo — passenger ke phone pe SMS jayega.

POST /api/v1/rides/:rideId/generate-otp
5
Passenger Se OTP Lo → Verify Karo

Passenger batayega 4-digit OTP. Driver app mein enter karo. OTP verify hone pe ride automatically in_progress ho jaati hai.

POST /api/v1/rides/:rideId/verify-otp LISTEN: ride:otp_verified
6
Drop Pe Pohoncho → Ride Complete Karo

Drop location pe pahunchne ke baad ride complete karo.

PATCH /api/v1/rides/:rideId/status {status: "completed"}
7
Payment Collect Karo (method pe depend karta hai)

Cash/UPI: collect-confirm API call karo. Wallet/Card: kuch nahi karna, auto-credit hogi.

POST /api/v1/rides/:rideId/collect-confirm
8
Summary Dekho → Next Ride Ka Intezaar

Trip summary screen dikhao. Rating bhi de sakte ho passenger ko.

GET /api/v1/rides/:rideId/driver-summary
💵 Cash Payment Flow — Details
Cash rides mein driver ke paas company ka paisa bhi hota hai. Ye platform fee deposit karni hoti hai — tab hi apna paisa milta hai.
Cash Ride — Pura Flow
1
Ride Complete Hoi

PATCH status=completed karo. Example: Fare = ₹120, Platform fee = ₹18, Driver share = ₹102

2
Cash Physically Lo Passenger Se

Passenger ₹120 cash deta hai. Driver ke paas ab ₹120 physically hai.

LISTEN: driver:cash_dues_updated — screen pe ₹18 pending dikhao
3
Collect Confirm API Call Karo

Driver button press kare: "Maine cash le liya". Backend mein dues track hoti hain, wallet mein paisa abhi nahi jaata.

POST /api/v1/rides/:rideId/collect-confirm {method: "cash"}
4
Platform Fee Deposit Karo

Driver ke paas ₹102 uska hai + ₹18 company ka. ₹18 deposit karna hai. Deposit ki ek limit hoti hai — exceed karne pe naye rides nahi milenge.

GET /api/v1/drivers/cash/balance — pending amount dekho POST /api/v1/drivers/cash/deposit — deposit submit karo
5
Earning Wallet Mein Aa Jaati Hai

Deposit submit karne ke baad ₹102 automatically wallet mein credit ho jaata hai.

Auto-deduction: Agar driver ka cash pending hai aur koi online ride complete karta hai, toh wo online earning se automatically cash dues kat jaate hain aur held earnings wallet mein release hoti hain.
📱 Personal UPI Flow
Personal UPI Ride — Flow
1
Ride Complete → Passenger Apne Driver Ka UPI Pe Pay Kare

Driver apna UPI QR/ID dikhata hai. Passenger directly driver ke UPI pe pay karta hai.

2
Payment Confirm Hone Pe Collect Confirm Karo

Driver ko payment receive hone ki confirmation milne ke baad collect-confirm API call karo.

POST /api/v1/rides/:rideId/collect-confirm {method: "personal_upi"}
3
Platform Fee Deposit Karo

Cash ki tarah hi — platform fee deposit karni hogi aur tab earning wallet mein aayegi.

POST /api/v1/drivers/cash/deposit
👛 Wallet / Card / Online UPI Flow
Online Payment — Driver Ke Liye Sabse Simple
1
Ride Complete Karo

PATCH status=completed. Payment already passenger ke wallet/card se deduct ho jaata hai — driver kuch nahi karta.

2
Earning Automatically Wallet Mein Aa Jaati Hai

Driver ko koi collect-confirm nahi karna. Backend automatically driver ka net earnings wallet mein credit karta hai.

LISTEN: driver:earnings_credited — wallet balance update karo
Bonus: Agar driver ke cash dues pending hain, toh online ride ki earning se pehle wo dues automatically cut ho jaate hain, aur held cash earnings bhi release ho jaati hain.

// Body nahi bhejni — rideId URL mein hoti hai

💰 Payment & Cash Settlement APIs
POST /api/v1/rides/:rideId/collect-confirm
Driver confirm karo ki cash/personal-upi payment collect ho gayi
📌 Kab: Ride complete status ke baad, jab payment method cash ya personal_upi ho.
Wallet/Card rides mein ye call nahi karna — automatic credit hoti hai.
Request Body
{
  "method": "cash"  // "cash" | "personal_upi"
}
Response
{
  "success": true,
  "message": "Collection recorded. Deposit ₹18 to receive your ₹102 earnings.",
  "data": {
    "rideId": 123,
    "method": "cash",
    "finalFare": 120,
    "platformFee": 18,
    "netEarnings": 102,
    "pendingDues": 18,    // Deposit karna hai
    "walletCredited": false  // Deposit ke baad credit hogi
  }
}
GET /api/v1/drivers/cash/balance
Driver ka pending cash dues aur cash stats dekho
📌 Kab: Home screen pe pending dues dikhane ke liye. Deposit screen open karne se pehle. Daily dekho.
Response
{
  "success": true,
  "data": {
    "pendingAmount": 54.00,        // Ye deposit karna hai (company ka paisa)
    "totalCashCollected": 340.00,  // Aaj tak kitna cash liya
    "totalDeposited": 286.00,      // Kitna deposit kiya
    "totalPlatformShare": 54.00,   // Same as pendingAmount (net)
    "cashLimit": 500.00,           // Max limit — exceed hone pe rides band
    "isLimitExceeded": false,
    "depositDueDate": "2024-01-03"  // Iss date tak deposit karo
  }
}
isLimitExceeded = true hone par driver ko naye rides nahi milenge. Immediately deposit karo.
